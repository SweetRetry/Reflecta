# 🪶 Design System

> **Stack**: Tailwind CSS v4 + shadcn/ui + motion  
> **Philosophy**: 现代 · 简约 · 克制 · 安静与创造力并存  
> **Priority**: Accessibility (A11y) & User Experience (UX)

---

## Ⅰ. 基础配置与主题

### A. 模式策略

- **主题模式**：支持 Light / Dark。
- **默认模式**：Dark。
- **开发指令**：
  > Code Agent 分析场景后决定模式（创作环境倾向 Dark，内容展示或营销页面可启用 Light）。
- 所有颜色定义必须基于 **统一 token**，不可独立硬编码。
- Shadcn 主题变量应为单一真源。

### B. 字体与排版

- **基础字体**：`Inter, sans-serif`
- **层级表达**：通过字重与字号区分，不依赖颜色。
- **正文颜色**：`text-foreground`
- **辅助文本**：`text-muted-foreground`
- **行距与阅读感**：以可读性优先，避免紧凑。

### C. 间距与留白

- **核心理念**：空间即节奏。
- **模块间距 (Section Gap)**：`space-y-12 ~ space-y-16`
- **卡片内边距 (Padding)**：默认 `p-6`，移动端可降至 `p-4`
- **规则**：宁宽勿挤，保证视觉呼吸感。

### D. 圆角与层次

- **圆角统一**：`rounded-xl`
- **阴影控制**：深色模式下优先使用 `border` 或 `bg` 层差，而非阴影。
- **层次表达**：通过背景层级（`bg-muted`, `bg-card`, `bg-popover`）体现。

---

## Ⅱ. 可访问性与对比度

- **语义化结构**：所有交互元素使用原生语义标签（`<button>`, `<nav>`, `<main>` 等）。
- **对比度标准**：遵守 WCAG AA
  - 普通文本 ≥ 4.5:1
  - 大号文本 ≥ 3:1
- **焦点可见性**：禁用默认 outline 时必须提供自定义可视焦点。

---

## Ⅲ. 状态与反馈

### A. 加载状态

- 使用 `Skeleton` 占位符。
- 风格：`bg-muted animate-pulse`，形状接近实际内容。
- 避免纯空白或仅转圈加载。

### B. 空状态

- 居中容器 + 图标 + 提示文案 + 主操作。
- 图标尺寸：`w-10 h-10`
- 说明文字：`text-muted-foreground`
- 主按钮：`variant="default"`

### C. Toast 与通知

- 所有轻量反馈使用 shadcn/ui 的 `Toast`。
- 状态色遵循语义：
  - Success → 绿色
  - Error → 红色
  - Warning → 黄色

---

## Ⅳ. 交互元素与图标

- **最小点击区域**：`44px × 44px` (`w-11 h-11`)
- **图标系统**：`lucide-react`
- **图标风格**：仅使用线性（outline）图标
- **图标尺寸**：`w-5 h-5`

---

## Ⅴ. 动效与交互节奏（Motion & Interaction Rhythm）

### A. 动效哲学

- 目的：传达“静中有动”的流畅感，不打断思考。
- 风格：**缓入缓出**、自然、克制。
- 动效存在感 < 设计存在感。

### B. 动效层级

| 层级 | 说明 | 时长 | 应用 |
|------|------|------|------|
| 微交互 (Micro) | 按钮 hover、图标旋转 | 150–200ms | `transition-all duration-200` |
| 结构变换 (Structural) | 卡片显隐、面板折叠 | 250–350ms | `motion.div` |
| 系统过渡 (System) | 页面、模态切换 | 400–600ms | `framer-motion` scene transition |

> **指令**：
> Code Agent 自动匹配动效层级。禁止无节奏状态切换。若需复杂转场，可引入 motion 库，但动画需轻盈、节制。

---

## Ⅵ. 视觉气质与语义延展（Visual Semantics & Aesthetic Tone）

### A. 视觉关键词

**Modern · Minimal · Controlled · Quietly Creative**

### B. 设计原则

1. **减少条框**：弱化边界，使用轻 border 或背景差异。
2. **留白主导**：分隔区块优先使用 `space-y-*` 而非线条。
3. **层级表达**：背景深浅体现结构，不靠阴影。
4. **视觉焦点**：通过排版节奏与位置组织视觉流。

### C. 创造力与安静的平衡

- 安静来自柔和对比与有呼吸的空间。
- 创造力通过微动效、字体张力与节奏变化体现。
- 不依赖饱和色或过强光感。

> **指令**：
> Code Agent 根据语义判断模块类型：
>
> - 内容区 → 使用 muted 层级
> - 创意或灵感区 → 允许 foreground 层级与微动画

---

## Ⅶ. 内容语气与文案调性（Tone & Voice）

### A. 品牌语气

| 属性 | 描述 |
|------|------|
| 理性 | 中性表达，避免感叹语。 |
| 启发 | 鼓励探索与创造。 |
| 克制 | 精简语言，去除多余修饰。 |
| 可信 | 语气平稳，传递可靠感。 |

### B. 文案规则

| 场景 | 用语 | 示例 |
|------|------|------|
| 空状态 | 提问式启发 | “准备好创作了吗？” |
| 错误提示 | 平静说明 | “保存遇到问题，请稍后重试。” |
| 成功提示 | 简短陈述 | “已完成。” |
| 加载状态 | 告知性 | “正在准备资源…” |

> **指令**：
> Code Agent 在渲染 Toast 或空状态时应验证文案语气，禁止使用命令式或情绪化词汇。

---

## Ⅷ. 响应式与密度策略（Responsive & Density System）

### A. 密度模式

| 模式 | 特征 | 场景 | Spacing 指令 |
|------|------|------|----------------|
| Comfort | 留白大、动效缓 | 创作、展示 | `space-y-12 p-8` |
| Balanced | 默认密度 | 通用 | `space-y-8 p-6` |
| Compact | 紧凑排布 | 控制台、面板 | `space-y-4 p-4` |

> **指令**：
> Code Agent 根据 viewport 与页面语义选择密度层级。

---

## Ⅸ. 系统总结

| 维度 | 定义 |
|------|------|
| **哲学** | 现代、简约、克制、安静与创造力并存 |
| **技术基础** | Tailwind CSS v4 + shadcn/ui + motion |
| **主题策略** | Token 统一，模式自适应 |
| **动效体系** | 三层级，节奏控制 |
| **视觉策略** | 留白优先，色彩克制，层次柔和 |
| **语气体系** | 理性启发、平静可信 |
| **执行准则** | Code Agent 分析场景 → 自动选择模式与密度 |
